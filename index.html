What Agents Nail

Clean runs share common traits. The task is unambiguous, single-concern, and requires no design judgment.

CSS & Layout Fixes

Padding, z-index, positioning, colors, dark mode. Agents are surgical with targeted style changes.

  • 1 commit Increase chat max-width to 760px
  • 1 commit Fix help tooltip cutoff

Removals & Deletions

The simplest task for an agent. "Remove X" or "Hide Y" almost never need iteration.

  • 1 commit Remove deep research tool
  • 1 commit Remove chat setup button
  • 1 commit Hide follow-up in view mode

Config & Feature Flags

Mechanical changes with clear patterns. Agent adds the flag, wires it up, done.

The 19 high-iteration PRs reveal four recurring patterns — each with a different fix.

HUMAN BEHAVIOR

SYSTEM CHANGE

When a PR touches routing + UI + state + i18n, agents need many rounds to get it right.

Building the Software Factory

Only 1 of 4 thrash causes requires changing human behavior. The other 3 are system problems with system solutions.

Change human behavior

Scope discipline

One prompt, one PR, one concern. Train the team to resist "while you're at it" — the single biggest thrash driver.

Change the system

Faster local design loop

Agents need to see what they built. Add screenshot review to the agent workflow so visual iteration happens in one pass, not 22 commits.

Auto-decompose multi-component tasks

Agent should break large features into sequential sub-tasks across frontend and backend. Build multi-repo support so agents can coordinate across boundaries.

Classify and fix error patterns

Go deep on the fix-on-fix spirals. Categorize the errors agents make, then build the fixes into agent defaults — lint before commit, type-check before push, validate i18n keys.

Eliminate pipeline noise

Auto-format before commit (kills 10% of noise). Move i18n post-merge (kills 18%). That's ~28% fewer commits with zero behavior change.
