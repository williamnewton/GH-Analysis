What Agents Nail

Clean runs share common traits. The task is unambiguous, single-concern, and requires no design judgment.

CSS & Layout Fixes

Padding, z-index, positioning, colors, dark mode. Agents are surgical with targeted style changes.

  • 1 commit Fix FAB overlap with classifier
  • 1 commit Increase chat max-width to 760px
  • 1 commit Fix help tooltip cutoff

Removals & Deletions

The simplest task for an agent. "Remove X" or "Hide Y" almost never need iteration.

  • 1 commit Remove deep research tool
  • 1 commit Remove chat setup button
  • 1 commit Hide follow-up in view mode

Config & Feature Flags

Mechanical changes with clear patterns. Agent adds the flag, wires it up, done.

  • 1 commit Add extended context limit flag
  • 2 commits Instrument AI context events
  • 1 commit Replace dark mode hex values

What Causes Thrash

The 19 high-iteration PRs reveal four recurring patterns that lead to wasted cycles.

1. Scope Creep

A PR starts as one thing, then more gets added. The worst offender: #92154 started as "add a button" and ballooned to 49 commits across 4 authors — popover, modal, switcher UI, z-index fixes.

2. Visual Design Iteration

Agents can't judge aesthetics. Tasks like "make it slap" or "update the look and feel" require many human-agent rounds. #94204 (SVG collage) needed 22 commits of repositioning and blur adjustments.

3. Multi-Component Features

When a PR touches routing + UI + state + i18n, agents need many rounds. These tasks should be decomposed into sequential, smaller prompts.

4. Fix-on-Fix Spirals

In high-thrash PRs, ~40% of substantive commits are fixes — the agent fixing what it just wrote. Each fix risks introducing new issues, creating a correction cascade.

Defaults to Build

01

Auto-format before commit

Agents should run the linter/formatter as their last step. Eliminates ~10% of all commits.

02

Post-merge i18n pipeline

Move Smartling translations out of PRs. Removes ~18% of commit noise and simplifies review.

03

Prompt templates

Pre-built templates for CSS fixes, feature flags, removals, and renames. These patterns are solved.

04

Scope gates at 5 commits

Alert when a PR crosses 5 substantive commits. Signal to split the task before thrash compounds.

05

Screenshot review for UI

Any task involving visual design should include an automated screenshot check before commit.

06

One agent per PR

Multi-author PRs correlate strongly with thrash. Enforce single-agent ownership to preserve context.

Agents are already writing most of our code. The leverage isn't in making them smarter — it's in giving them better-scoped tasks and less noisy pipelines.

The difference between a 1-commit PR and a 49-commit PR is almost never the agent. It's the prompt, the scope, and the system around it.
