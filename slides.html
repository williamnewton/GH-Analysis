What Causes Thrash

The 19 high-iteration PRs reveal four recurring patterns — each with a different fix.

Change human behavior Change the system

1. Scope Creep

HUMAN BEHAVIOR

A PR starts as one thing, then more gets added. #92154 started as "add a button" → 49 commits across 4 authors.

#94401 AI Controls polish — 36 commits, 5 authors

#92848 Dashboard popover adjustments — 28 commits, 5 authors

#94097 Splash page look & messaging — 23 commits, 5 authors

2. Visual Design Iteration

SYSTEM CHANGE

Agents can't judge aesthetics. "Make it slap" required 22 commits of repositioning and blur adjustments on #94204.

#97936 Opportunities view cleanup — 23 commits of redesign

#92945 Shared agents UI post-launch — 21 commits, button tweaks

#94400 Debug panel dark mode — 14 commits, 7 fixes

3. Multi-Component Features

SYSTEM CHANGE

When a PR touches routing + UI + state + i18n, agents need many rounds to get it right.

#97129 Opportunities manage page — 25 commits, routing + tabs + flags

#93246 Session Replay agents popover — 19 commits, modal + prompts + z-index

#96944 Opportunity detail page — 13 commits, extract route + layout

4. Fix-on-Fix Spirals

SYSTEM CHANGE

In high-thrash PRs, ~40% of substantive commits are fixes — correction cascades where each fix risks new issues.

#92198 Share modal — 16 of 15 substantive commits were fixes

#92848 Agents popover — 13 fix commits out of 20 substantive

#93683 Ask Amplitude warning — 4 fixes out of 6 substantive

Building the Software Factory

Only 1 of 4 thrash causes requires changing human behavior. The other 3 are system problems with system solutions.

Change human behavior

Scope discipline

One prompt, one PR, one concern. Train the team to resist "while you're at it" — the single biggest thrash driver.

Change the system

Faster local design loop

Agents need to see what they built. Add screenshot review to the agent workflow so visual iteration happens in one pass, not 22 commits.

PROTOTYPE BUILT  Playwright visual regression tool in visual-regression/ — auto-screenshots pages, pixel-diffs against baselines, exits non-zero on visual change. Ready to integrate into agent workflow.

Auto-decompose multi-component tasks

Agent should break large features into sequential sub-tasks across frontend and backend. Build multi-repo support so agents can coordinate across boundaries.

Classify and fix error patterns

Go deep on the fix-on-fix spirals. Categorize the errors agents make, then build the fixes into agent defaults — lint before commit, type-check before push, validate i18n keys.

Streamline prompt → PR

The core loop is: describe what you want, get a link back. Every extra step — switching tools, finding the right file, waiting for CI — adds friction that compounds. Make the happy path one message to one merged PR.
